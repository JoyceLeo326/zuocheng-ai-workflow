# 做成■ 指标与实验

## 一、北极星指标

**每周有效交付数**：一周内完成、通过核验并能被真实提交的任务结果数量。

一次有效交付需要同时满足：

- 有明确任务定义。
- 达到约定格式与时间要求。
- 关键判断有来源支撑。
- 用户完成过最终核验。
- 结果确实用于课堂、研究、比赛或答辩等现实场景。

播放课程、生成初稿、导出文件都不是有效交付。

## 二、核心漏斗

```text
看见任务内容
  ↓ 内容→页面点击率
进入任务页面
  ↓ 页面→任务卡领取率
领取任务卡
  ↓ 任务定义完成率
开始第一课
  ↓ 七天首次交付率
完成首次交付
  ↓ 主动分享率
发布过程或邀请同伴
  ↓ 七天第二任务启动率
再次使用工作流
```

## 三、事件定义

| 事件 | 触发条件 | 必要属性 |
|---|---|---|
| `content_viewed` | 任务内容进入可视区域 ≥ 3 秒 | 内容母题、渠道、任务类型 |
| `task_page_opened` | 打开具体任务页面 | 来源内容、页面版本 |
| `task_card_claimed` | 成功保存任务卡 | 任务类型、入口 |
| `task_defined` | 四项必要字段齐全 | 听众、格式、截止时间、标准 |
| `evidence_bound` | 证据绑定原始位置 | 来源类型、可信等级 |
| `structure_completed` | 形成完整叙事结构 | 页面数、结构类型 |
| `draft_generated` | 生成可编辑初稿 | 生成次数、使用模型 |
| `verification_completed` | 核验项全部处理 | 问题数量、修改数量 |
| `deliverable_exported` | 导出交付包 | 文件类型、任务耗时 |
| `delivery_confirmed` | 用户确认用于现实提交 | 提交场景、结果状态 |
| `process_shared` | 发布完成过程或邀请同伴 | 分享渠道、内容类型 |
| `second_task_started` | 七天内创建第二项任务 | 原任务类型、新任务类型 |

## 四、指标口径

### 任务开始率

```text
完成任务定义的人数 / 领取任务卡的人数
```

同一用户七天内多次编辑只计一次。任务定义需包含四项必要字段。

### 首次交付率

```text
七天内确认现实提交的人数 / 开始第一课的人数
```

仅导出文件不计入，需要用户确认已用于现实任务并完成核验。

### 主动分享率

```text
交付后七天内发布过程或邀请同伴的人数 / 完成首次交付的人数
```

由系统自动生成但未实际发送的分享卡不计入。

### 第二任务启动率

```text
交付后七天内创建新任务的人数 / 完成首次交付的人数
```

该指标衡量工作流是否被迁移，而不仅是用户对一次结果是否满意。

## 五、验证期测算与目标

以下数字用于首批验证期的容量测算与决策门槛，不代表既有运营成绩；实际判断以统一埋点回收的数据为准。

| 指标 | 基准假设 | 验证目标 | 判断 |
|---|---:|---:|---|
| 页面→任务卡领取率 | 31% | ≥ 38% | 任务表达是否清楚 |
| 任务定义完成率 | 54% | ≥ 70% | 首步负担是否合理 |
| 七天首次交付率 | 32% | ≥ 45% | 工作流是否能推进完成 |
| 主动分享率 | 18% | ≥ 25% | 结果是否值得表达 |
| 第二任务启动率 | 暂无 | ≥ 20% | 方法是否可以迁移 |

小样本期同时报告人数与比例，避免比例掩盖基数。

## 六、实验卡模板

每轮实验只允许一个主变量：

```text
问题：哪一层流失最大？
证据：数据 + 用户原话
假设：如果改变 X，Y 会提升，因为 Z
变量：本轮唯一改动
主指标：用于做决策的一个指标
护栏：不能明显下降的指标
时间：开始与结束
阈值：继续、修改、停止的条件
结果：数字、口径与异常
决定：保留 / 重做 / 放弃
```

## 七、首批实验队列

### 实验一：工具表达 vs 任务表达

- 假设：具体任务输入与交付结果比工具功能更能推动领取。
- 变量：落地页首屏标题与主视觉。
- 主指标：页面→任务卡领取率。
- 护栏：进入页面后的停留时间不下降 20% 以上。
- 阈值：B 版相对提升 ≥ 20% 且样本达到最低观察量，保留 B。

### 实验二：空白任务卡 vs 完整填写结果

- 假设：真实填写结果能够减少用户对字段含义的不确定。
- 变量：任务卡旁是否展示完整填写结果。
- 主指标：任务定义完成率。
- 护栏：填写时长不增加。

### 实验三：长提示词 vs 四字段输入

- 假设：四个可选字段比长提示词更容易完成，也更容易修改。
- 变量：输入方式。
- 主指标：首步完成率。
- 护栏：输出被用户保留的比例。

### 实验四：结果晒单 vs 过程修改

- 假设：展示关键修改比只展示漂亮结果更能推动高意向用户开始。
- 变量：内容叙事结构。
- 主指标：内容→任务页面点击后开始率。
- 护栏：收藏率。

## 八、定性反馈编码

用户原话统一归入五类：

1. **不知道做什么**：任务定义问题。
2. **不知道信什么**：来源与核验问题。
3. **不知道怎么排**：结构问题。
4. **不知道怎么改**：反馈与编辑问题。
5. **做完不会复用**：沉淀问题。

每条原话需要同时记录：用户所在步骤、原始任务、当时行为、解决方式和是否继续。只记录“满意/不满意”无法指导下一版。

## 九、周复盘输出

每周五形成一页决策记录：

- 本周有效交付数
- 漏斗各层人数与转化率
- 最大流失层
- 三条代表性用户原话
- 本周修改与结果
- 下周唯一主实验
- 明确不做的三件事

复盘的目的不是汇报忙碌，而是决定下一周把哪一个 □ 推进为 ■。
