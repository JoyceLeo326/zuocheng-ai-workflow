# 做成■ 内容系统

## 一、内容任务

内容不负责证明“我们懂很多 AI”，而是帮助用户完成三次认知变化：

1. 从“我还缺一个更强工具”转向“我缺的是明确任务与完成路径”。
2. 从“AI 一键生成”转向“AI 加速、人做判断、过程可核验”。
3. 从“等有空再学”转向“带一项真实任务，今天推进一格”。

核心主张保持不变：

> 别再收藏工具，先做成一件事。

## 二、内容母题

### 母题一：收藏很多，仍然没开始

识别信号：收藏夹增长、反复注册、在空白文档前搜索新教程。

可用切口：

- “你收藏的第 37 个 AI 工具，今天能替你交哪份作业？”
- “教程看了 2 小时，为什么第一页还没写？”
- “换工具不是进度，任务状态变化才是。”

引导动作：领取一张具体任务卡，不领取宽泛课程大纲。

### 母题二：输入与交付之间缺一条路

识别信号：资料很多、提示词很长、生成结果无法直接使用。

可用切口：

- “25 页资料如何压成 3 页能讲的汇报？”
- “别让 AI 直接总结全文，先找 6 条能支撑结论的依据。”
- “一页一个结论，比一页五个知识点更难，也更重要。”

引导动作：进入工作台，完成定义、证据、结构中的一个节点。

### 母题三：流畅不等于可靠

识别信号：数字没有出处、引用不存在、相关关系被写成因果。

可用切口：

- “看起来最专业的那句话，可能最需要回查。”
- “AI 可以写得很顺，但它不替你承担引用责任。”
- “三步核验：回到原文、检查边界、删掉过度判断。”

引导动作：保存核验表，检查手上正在修改的内容。

### 母题四：一次完成，下一次重来

识别信号：每项任务都从搜索教程开始，优秀做法没有沉淀。

可用切口：

- “做完之后，留下的不能只有一个文件。”
- “把这次的任务卡、证据结构和核验表一起保存。”
- “真正省时间的不是生成，而是第二次不用重来。”

引导动作：保存任务模板，并在七天内启动第二项任务。

## 三、内容结构公式

### 任务对照型

```text
一个具体输入
→ 一个具体卡点
→ 一个关键判断
→ 一个可见结果
→ 一个立即动作
```

例：

```text
25 页调研资料
→ 信息太多，不知道先讲什么
→ 先写听众要回答的三个问题
→ 得到“问题—发现—建议”三页结构
→ 下载任务卡，写出自己的三个标题
```

### 错误纠正型

```text
常见做法
→ 为什么会失败
→ 只改一个节点
→ 修改前后对照
→ 核验方式
```

### 过程记录型

```text
原始待办 □
→ 中间最难的判断
→ 删除或修改了什么
→ 最终交付 ■
→ 下次保留的方法
```

## 四、平台分工

| 触点 | 核心任务 | 适合内容 | 成功信号 |
|---|---|---|---|
| 短视频 | 建立问题共鸣 | 空白文档、任务转化、修改过程 | 完播、主页访问 |
| 图文 | 解释方法并促使收藏 | 步骤卡、错误对照、核验清单 | 收藏、任务卡领取 |
| 社群 | 推动当天动作 | 截止提醒、结构互评、答疑 | 当日完成率 |
| 直播 | 现场证明能做成 | 真实资料拆解、交付诊断 | 开始第一课 |
| 邮件/通知 | 拉回中断用户 | 未完成节点提醒、结果回顾 | 回流完成率 |

## 五、三张海报的传播策略

### 海报一：打断收藏惯性

- 主标题：你收藏的第 37 个 AI 工具，今天能替你交哪份作业？
- 场景：内容流首触达。
- 目标：让用户对“收藏=进步”产生怀疑。
- 动作：进入 7 天任务页。

### 海报二：用结果建立确定性

- 主标题：25 页资料 → 3 页汇报。
- 支撑：38 分钟做出可讲版本。
- 场景：用户已对问题有共鸣，但仍怀疑课程是否具体。
- 目标：展示输入、时间与交付边界。
- 动作：打开第一课工作台。

### 海报三：传递系统方法

- 主标题：开学不重学工具，直接跑通任务。
- 场景：系统课转化。
- 目标：说明不是一次性技巧，而是一套可迁移工作流。
- 动作：查看 18 课路径。

## 六、30 秒视频脚本

| 时间 | 画面 | 文字/旁白 | 传播目的 |
|---|---|---|---|
| 0–3 秒 | 手指连续收藏工具合集 | 收藏夹已经 237 条 | 识别行为 |
| 3–7 秒 | 切换到空白文档 | 但下周的汇报，还是一个字没写 | 放大落差 |
| 7–12 秒 | 25 页资料与 3 页汇报写入任务卡 | 先定义要交什么 | 给出第一个判断 |
| 12–20 秒 | 标记 63.4% 与来源页码 | AI 可以加速，结论必须能核验 | 建立可信感 |
| 20–27 秒 | 问题、发现、建议三页依次完成 | 一页一个结论 | 展示可见结果 |
| 27–30 秒 | □ 变为 ■ | 别再收藏工具，先做成一件事 | 促使行动 |

## 七、社群文案原则

### 每条消息只做一件事

禁止在一条消息中同时要求“学习、打卡、分享、反馈”。每个触点必须有唯一动作与明确截止时间。

### 先给判断标准，再催完成

“记得交作业”不能解决卡点。需要告诉用户什么叫完成，例如：

> 今晚 22:00 前发三行页面标题。其他同学只看标题，能复述你的完整故事，就算完成。

### 晒修改，不只晒结果

优先呈现删掉了什么、核验了什么、为什么换了结构。过程更能证明方法，也更容易被处在同一任务的人迁移。

## 八、内容核验清单

发布前逐项确认：

- 标题是否指向一个真实任务，而不是泛化的 AI 焦虑。
- 是否给出输入、动作或交付中的至少两个具体信息。
- 所有数字是否注明口径与来源。
- 是否把相关关系误写成因果关系。
- 是否暗示一键完成本应由用户判断的工作。
- 行动按钮是否与内容承诺完全一致。
- 用户进入下一步后，是否能在十分钟内获得第一个结果。
